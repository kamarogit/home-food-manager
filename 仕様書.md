# 家庭用食材管理システム 仕様書 v1.0

## 1. 概要

本システムは、家庭内で保有している食材をデータベースで管理し、WebアプリおよびMCPサーバ経由で登録・更新・削除・検索できるようにすることを目的とする。

本システムは以下の4要素で構成する。

* 食材情報を保持するデータベース
* 食材データを操作するAPI
* APIを利用して食材を管理するWebアプリ
* APIを利用して食材を操作するMCPサーバ

初期段階では、家庭内での個人利用または少人数利用を想定し、シンプルで保守しやすい構成を優先する。

---

## 2. 目的

本システムの目的は以下の通り。

* 家にある食材を一覧で把握できるようにする
* 残量や保存場所、期限情報を記録して管理しやすくする
* 食材の使い忘れや買い忘れを減らす
* MCP経由でLLMなどの外部ツールから食材情報を参照・更新できるようにする
* 将来的に買い物支援や献立提案などへ拡張しやすい基盤を作る

---

## 3. 想定ユースケース

### 3.1 基本ユースケース

* 冷蔵庫、冷凍庫、棚などにある食材を登録する
* 食材の残量状態を更新する
* 食材の保存場所を確認する
* 消費期限や開封日を記録する
* 不要になった食材を削除する
* 補充が必要な食材を確認する

### 3.2 MCP利用ユースケース

* 「卵はあるか」を検索する
* 「冷蔵庫にある食材を一覧表示する」
* 「購入必要な食材を一覧表示する」
* 「牛乳を購入必要に更新する」
* 「豆腐を追加する」
* 「もう使わない食材を削除する」

## 4. システム構成

### 4.1 構成要素

#### DB

食材マスタおよび在庫情報を保持する。

#### API

WebアプリおよびMCPサーバから利用されるデータ操作用インターフェースを提供する。

#### Webアプリ

ユーザーがブラウザ上で食材を登録・編集・削除・検索できるUIを提供する。

#### MCPサーバ

LLMなどのクライアントからAPIを呼び出し、食材情報の追加・更新・削除・検索を可能にする。

### 4.2 初期実装候補

* Backend: FastAPI
* DB: SQLite
* MCP: FastMCP
* Frontend: シンプルなWebアプリ

## 6. 用語定義

### 6.1 食材マスタ

在庫として登録可能な食材の定義情報。食材名の表記ゆれを防ぐため、在庫登録時は自由記載ではなく食材マスタから選択する。

### 6.2 在庫

現在管理対象となっている家庭内の食材情報。食材マスタを参照して登録する。

※ APIパスやMCPツール名では `ingredients` という語を使うが、本仕様では「在庫」を指す。

### 6.3 残量状態

食材の残り具合を表す状態値。数値や重量ではなく、以下の3状態で管理する。

* 多い
* 少ない
* 購入必要

### 6.4 購入必要

在庫がなくなった、または少なくなり、今後補充したい状態を表す。

## 7. データ設計

### 7.1 設計方針

* 食材名は自由記載ではなくマスタ制とする
* 在庫登録時は食材マスタから対象食材を選択する
* 残量は数値ではなく状態で管理する
* 詳細な個数や重量は原則管理しない
* 必要な補足はメモ欄に自由記載できるようにする

### 7.2 テーブル構成

#### 7.2.1 カテゴリテーブル

| カラム名       | 型        | 必須 | 説明     |
| ---------- | -------- | -: | ------ |
| id         | integer  |  ○ | 主キー    |
| name       | string   |  ○ | カテゴリ名  |
| is_active  | boolean  |  ○ | 使用中フラグ |
| sort_order | integer  |  ○ | 表示順    |
| created_at | datetime |  ○ | 作成日時   |
| updated_at | datetime |  ○ | 更新日時   |

#### 7.2.2 食材マスタテーブル

| カラム名       | 型        | 必須 | 説明     |
| ---------- | -------- | -: | ------ |
| id         | integer  |  ○ | 主キー    |
| name       | string   |  ○ | 食材名    |
| category_id| integer  |    | カテゴリID |
| is_active  | boolean  |  ○ | 使用中フラグ |
| created_at | datetime |  ○ | 作成日時   |
| updated_at | datetime |  ○ | 更新日時   |

#### 7.2.3 在庫テーブル

| カラム名                 | 型        | 必須 | 説明              |
| -------------------- | -------- | -: | --------------- |
| id                   | integer  |  ○ | 主キー             |
| ingredient_master_id | integer  |  ○ | 食材マスタID         |
| quantity_status      | enum     |  ○ | 多い / 少ない / 購入必要 |
| storage_location     | string   |    | 保存場所            |
| expiry_date          | date     |    | 期限日             |
| opened_date          | date     |    | 開封日             |
| note                 | text     |    | メモ              |
| created_at           | datetime |  ○ | 作成日時            |
| updated_at           | datetime |  ○ | 更新日時            |

### 7.3 カテゴリ

カテゴリはカテゴリテーブルで管理し、食材マスタ登録時に選択する。初期候補は以下の通り。

* 野菜
* 肉
* 魚
* 乳製品
* 調味料
* 飲料
* 冷凍食品
* その他

### 7.4 保存場所

初期版では以下の候補値を想定する。

* 冷蔵
* 冷凍
* 野菜室
* 常温
* 棚
* その他
* 未設定

実装時には固定候補とするか、マスタ管理とするかを選択する。

## 8. 在庫運用ルール

### 8.1 在庫登録

* 在庫登録時は食材マスタから食材を選択する
* 食材名の自由入力では登録しない
* 残量状態は必須入力とする
* 保存場所、期限日、開封日、メモは任意入力とする

### 8.2 在庫切れ時の扱い

* `なし` という状態は持たない
* 在庫がなくなった場合は以下のどちらかとする

  * 今後も補充したい場合は `購入必要` に更新する
  * 補充予定がなく管理対象から外す場合は削除する

### 8.3 メモ

以下のような補足情報をメモ欄に記載できる。

* 数量の補足
* 状態の補足
* 購入時の注意事項
* 調理用メモ
* その他の自由記述

例:

* 2本あり
* 半分くらい残っている
* カレー用
* 開封済み
